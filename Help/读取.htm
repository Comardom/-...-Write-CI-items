Q：“读取”是什么意思？
A：“读取”操作指的是在您按下按钮【一键显示配置文件中配置内容】（这时未经过解密）或【解密后显示配置文件中配置内容】（这时经过解密）时对程序同文件夹下的配置文件进行读取并显示在输入框内。如果您对解密感兴趣，请参阅页面：帮助_关于解密。

Q：可以一条一条内容读取吗？
A：不可以。原因见页面：帮助_关于 UI 。

Q：输入框中输入了内容以后再点击读取按钮（其二之一）会保留哪部分内容？不保留的数据会怎么处理？
A：会保留您想显示的配置文件中的数据，输入框中的内容不做保留。

Q：按下了读取按钮（其二之一）之后输入框内不完全出现内容并且不出现乱码怎么办？
A：请按照以下顺序进行检查：
        1. 检查您要读取的配置文件是否和程序处于同一个文件夹。{ 若不在，请在备份好您当前的数据并关闭窗口后移动程序文件到配置文件所属的文件夹下，或者在确保已经关闭配置文件的文本显示窗口及没有其余进程读取此配置文件的情况下将配置文件移动进程序文件所在文件夹（此处建议您在转移之前先行进行备份） }
        2. 检查程序文件下是否有其他的配置文件与您所要读取的文件重名。
        3. 检查高级选项内的填充是否为您的预设。
        4. 如果您是解密后读取的文件（文件已经被加密过的），请立即检查您是否对原来未加密的数据有备份，如果有备份，请对备份文件进行读取确保文件的完整性（如果备份文件中的数据是完整的，请参阅以下页面寻找解决办法：帮助_关于加密）；如果您确定没有保存的备份数据，那么您的数据有可能已经永久丢失，请寻找专业的付费数据恢复方式。{为确保尽量不出现吞数据的问题，建议您在每次执行写入操作的时候都进行备份（使用程序内置的备份功能），并且在写入操作完成后点击读取按钮（如果您未进行加密，请按按钮【一键显示配置文件中配置内容】；如果您已经进行加密，请按按钮【解密后显示配置文件中配置内容】）检查数据是否已经完全写入。注意：请尽量在已经备份文件的情况下执行“读取”操作，读取操作会覆盖原本内容，存在使得输入框变空的可能性。}

Q：按下了读取按钮（其二之一）之后输入框内不完全出现内容并且出现乱码怎么办？
A：A：请按照以下顺序进行检查：

        3. 检查高级选项内的填充是否为您的预设。
        4.检查您的文件是否存在双重加密。{如果确认是有双重加密，请先进行逐步备份再一步步解密读取}
        5.  如果您是解密后读取的文件（文件已经被加密过的），请立即检查您是否对原来未加密的数据有备份，如果有备份，请对备份文件进行读取确保文件的完整性（如果备份文件中的数据是完整的，请参阅以下页面寻找解决办法：帮助_关于加密）；如果您确定没有保存的备份数据，那么您的数据有可能已经永久丢失，请寻找专业的付费数据恢复方式。{为确保尽量不出现吞数据的问题，建议您在每次执行写入操作的时候都进行备份（使用程序内置的备份功能），并且在写入操作完成后点击读取按钮（如果您未进行加密，请按按钮【一键显示配置文件中配置内容】；如果您已经进行加密，请按按钮【解密后显示配置文件中配置内容】）检查数据是否已经完全写入。注意：请尽量在已经备份文件的情况下执行“读取”操作，读取操作会覆盖原本内容，存在使得输入框变空的可能性。}

Q：保存了备份文件以后要如何读取？
A：程序内置的备份功能仅会储存未经加密的数据。如果您要读取的配置文件是"config_copy.ini",则请将程序移动到此文件所处的文件夹下，或在确保此配置文件未被编辑中且未被任何程序占用的情况下将文件移动到程序所在文件夹下，然后再在高级选项里勾选选择框【读取拷贝文本】，最后回到常规配置点击按钮【一键显示配置文件中配置内容】。如果您要读取的配置文件的名字很长，含有很多内容,则请把文件从原本的文件夹直接拖入到高级选项里的黑色线框组成的区域中，程序会直接读取您的数据。
